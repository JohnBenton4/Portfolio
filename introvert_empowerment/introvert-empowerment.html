Introvert Empowerment

Overview

Vision

Our goal with this project was not only to build an amazing app, but it was also to truly help people who have a legitiment desire to improve a part of their life. Many introverts, like myself, would love to be a little more outgoing from time to time. My team build this app to provide ideas on how to do just that.

Design

Being our capstone project, we felt it was extremely important to have a presentable UI along with a functionally top notch back end. We took everything we've learn through out our course and packed as much of it into this app as we could. A few notes about design:

  • The front end utilizes an ecouraging message of the day which comes from an API
  • The three main sections for user imporovement consist of, tasks which he or she can choose from, a task mamnger, and phone notifications
  • Users can create a profile which they can log in from any device and save information to their profile

Bootstrap was the main CSS framework used on this project.

Logistics

  • Development: October 8 - 27
  • My roles: SQL set up, login credential and bcrypt, API set up, user profile

Results

Features

This project was very challenging. We completed all the base requirements and moved on to extra features that made this app standout. Here are some of the features within our code:

  • API call with a timer to populate only one per 24 hours
  • User profiles that are password protected and can hold user saved data
  • The ability to link your app to your phone to receive text reminders to complete tasks
  • Establishing a connection between multiple many to many tables for each user

Project Reflection

This was the first project that I worked on by myself. It was much easier to formulate a plan a strategy for this project and time management was much less complicated. There were many moments in the this project where I really had to stretch my mind and search for information in resources. The most difficult portion was getting the string from the first API fetch to populate in the second API fetch. I am very pleased with the functional results and look forward to revisiting the project to add a small amount of visual interest.
